# 复试项目介绍

## 项目标题

基于时序知识图谱的慢性病辅助诊疗系统设计与实现

## 1 分钟介绍

我基于时序知识图谱推理模型 CTpath，设计并实现了一个慢病辅助诊疗系统。系统底层使用 CHRONIC 数据集和训练好的时序图谱模型进行病情推演，中间层使用 FastAPI 封装登录、患者、时间线和预测接口，上层使用 Vue 构建医生工作台，实现了患者检索、病程时间线展示、T+1 风险预测和辅助诊疗建议。在样本不足的情况下，系统还会自动切换到相似病例辅助模式，以保证系统在真实场景下的可用性。

## 复试重点亮点

- 我完成了从模型复现到系统落地的完整链路
- 我将通用时序知识图谱模型迁移到了慢病场景
- 我设计了 MySQL 患者事件表并封装成 FastAPI 服务
- 我实现了可演示的医生端 Web 工作台
- 我考虑了低样本场景下的降级策略，而不是只展示模型最理想的结果

## 可量化结果

- `CHRONIC` 数据集：`MRR ≈ 0.345`
- `Hits@1 ≈ 0.232`
- `Hits@10 ≈ 0.515`
- 系统支持登录、检索、时间线、预测和建议闭环演示

## 面试常见问题回答方向

### 如果老师问：这是不是只是复现论文？

回答重点：

- 不是只停留在模型复现
- 我把模型迁移到了慢病场景
- 我增加了数据库、接口和前端系统
- 我考虑了真实使用场景下的低样本与降级问题

### 如果老师问：样本少怎么办？

回答重点：

- 数据充分时走时序图谱模型推理
- 数据不足时切到相似病例辅助建议
- 这样系统在真实场景中更稳定，也更符合医疗应用的保守要求

### 如果老师问：你的工作量体现在哪？

回答重点：

- 数据构建：把患者事件转成时序图谱格式
- 模型迁移：训练 CHRONIC 数据并接入推理
- 后端服务：封装完整接口
- 前端展示：实现医生工作台
- 系统联调：打通数据库、模型和界面
