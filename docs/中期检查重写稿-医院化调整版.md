# 中期检查重写稿

根据指导教师关于“中期阶段以系统实现为主、任务书内容仅作参考、应结合实际进展进行调整”的意见，课题当前已不再单纯围绕原任务书中的模型复现与实验指标展开，而是结合项目实际推进情况，将中期阶段的工作重点调整为“面向慢病辅助诊疗与随访协同场景的医院化原型系统设计与实现”。任务书中的研究内容仍作为总体方向参考，但在本阶段更强调系统原型搭建、模块结构优化、前后端联调、岗位分工体现以及与真实医院业务流程的贴近程度。

前期工作已完成课题背景梳理、总体方案设计、数据流与功能链路分析，并在此基础上逐步形成以“患者档案”为中心、以“临床评估、档案治理、随访协同、治理中心”为主线的系统结构。当前系统后端基于 FastAPI 实现，前端基于 Vue 3 实现，已经具备完整的账号登录、患者列表查询、患者详情查看、患者建档与更新、结构化事件补录、联系记录录入、随访任务处理、流程状态展示、模型推理调用、辅助建议生成等主要功能，能够形成可演示的业务闭环。

从系统实现情况看，当前阶段已完成较多核心模块。后端方面，系统已经支持 MySQL 模式与演示数据模式双轨运行，完成了患者档案、结构化事件、随访任务、联系记录、门诊任务、系统治理等接口的统一接入；在认证层面，已从原有的简单明文校验升级为密码哈希校验，并逐步引入角色权限控制和档案审计能力。数据模型方面，患者档案字段已从原有的演示型结构扩展为更接近医院场景的结构，新增了头像、联系电话、紧急联系人、身份证脱敏号、医保类型、建档科室、责任医生、责任护士或管理师、过敏史、家族史、MRN、建档来源、档案状态、知情同意状态等字段，使系统不再停留在简单示例层面，而能够体现医院档案管理与慢病管理场景中的基础信息要求。

前端方面，系统已不再是单一页面的简单组合，而是逐步按照医院信息系统的业务域思路进行组织。当前已形成医生、护士、档案员三类角色导向界面，不同岗位登录后看到的默认入口、可见模块、首页重点内容和主要任务入口均不相同。医生端重点体现临床工作台，围绕接诊队列、风险研判、临床预警、门诊工作摘要、风险分组和档案联查展开；护士端重点体现随访与联系记录处理，形成面向班次任务的随访工作台；档案员端则围绕建档、MRN 维护、档案质控、资料补录和主索引治理展开。与此同时，系统已初步形成治理中心页面，用于展示模块目录、系统健康状态、主索引预警和维护概览，使整个系统从“功能演示界面”逐步过渡到“具有岗位分工与治理意识的医院化原型系统”。

在医院化改造方面，本阶段还参考了开源医院信息系统 `openhis` 的模块组织方式，对现有项目进行了有针对性的学习和吸收。学习后可以看出，较完整的医院系统通常不是简单按页面划分，而是按业务域组织为基础平台、统一认证、临床协同、患者主数据、病案管理、病案质控、接口治理等子系统。结合本课题的规模与定位，项目没有直接照搬完整 HIS 的大体量结构，而是在保留当前技术路线和已有风格的前提下，重点吸收其“按业务域组织系统”的思路，将本项目现有能力重新归纳为临床评估域、档案治理域、随访协同域和治理中心等模块。这一调整使项目在整体结构、角色分工和模块表达上更加贴近真实医院系统，也与指导教师提出的“加强系统实现、突出工程化和场景化”的要求保持一致。

从阶段性成果来看，课题目前已经完成了从“算法与数据准备”为主向“系统实现与医院化改造”为主的转段。一方面，原有模型与数据处理工作仍得到保留，系统能够基于患者历史结构化事件进行预测与辅助建议生成，并作为临床工作台中的支撑能力存在；另一方面，中期阶段的主线已经明确转向系统实现，即如何让模型、档案、随访、门诊任务和治理信息在同一原型系统中形成可展示、可交互、可扩展的业务流程。这说明课题已从单纯的模型实验型项目，逐步发展为“面向慢病辅助诊疗与随访协同的研究型医院化原型系统”。

当前仍存在的主要不足主要体现在以下几个方面。首先，虽然系统已经开始体现岗位边界和模块分域，但前端主调度和后端部分业务逻辑仍然较为集中，模块边界还需要进一步清晰，尤其是档案、随访、治理等能力仍有继续拆分和收敛的空间。其次，系统虽然已引入角色权限、主索引治理、档案质控和联系记录等内容，但整体仍处于轻量级原型阶段，与真实医院系统相比，在字段级权限、完整审计日志、会话令牌治理、接口管理和治理闭环方面仍有差距。再次，模型和实验部分虽然已经具备支撑作用，但其数据来源、实验覆盖和与真实临床流程的贴合程度仍有限，因此当前项目更适合定位为研究型原型系统，而不宜直接表述为可投入真实医院生产环境的成熟系统。

下一阶段将继续围绕系统实现主线推进工作。具体包括：进一步优化医生端、护士端、档案员端三类岗位工作台，继续强化不同角色的首页结构、任务流程和操作边界；继续完善档案治理能力，将患者主索引、MRN 维护、知情同意、档案质控和审计链进一步结合；继续拆分后端逻辑，提升档案、随访、任务、治理等模块的结构清晰度和可维护性；继续增强移动端随访流程，使其更贴近实际电话随访与外出随访场景；同时保留模型预测与建议生成功能，作为系统智能化支撑能力，用于支撑论文后续系统实现章节和方法章节的衔接。总体来看，课题中期阶段已顺利完成从研究准备到系统实现与医院化改造的过渡，具备了继续深化开发、完善论文内容并支撑后续答辩展示的基础。
