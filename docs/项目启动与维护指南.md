# 项目启动与维护指南

## 1. 环境准备

项目目录默认位于 `E:\CTpath-master`，建议使用两个终端分别启动后端和前端。

推荐环境：

- Python 3.10+
- Node.js 18+
- npm 9+
- 可选：MySQL 8.x
- 可选：conda 环境 `ctpath`

## 2. 启动后端

### 2.1 使用演示模式

如果暂时不连接 MySQL，后端会自动使用内置演示数据。

```powershell
conda activate ctpath
cd E:\CTpath-master
uvicorn app.main:app --host 127.0.0.1 --port 8000 --reload
```

启动后可访问：

- 接口文档：`http://127.0.0.1:8000/docs`
- 健康检查：`http://127.0.0.1:8000/api/health`

### 2.2 使用 MySQL 模式

先建库，例如：

```sql
CREATE DATABASE ctpath CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
```

然后依次导入：

1. `app/mysql_schema.sql`
2. `app/mysql_seed_demo.sql`

导入完成后，在启动后端前设置数据库连接：

```powershell
$env:CTPATH_DB_URL="mysql+pymysql://root:你的密码@127.0.0.1:3306/ctpath?charset=utf8mb4"
```

再执行：

```powershell
conda activate ctpath
cd E:\CTpath-master
uvicorn app.main:app --host 127.0.0.1 --port 8000 --reload
```

如果 `api/health` 返回 `mode: mysql`，说明已切换到数据库模式。

## 3. 启动前端

新开一个终端：

```powershell
cd E:\CTpath-master\frontend
cmd /c npm install
cmd /c npm run dev
```

启动后访问：

- 前端地址：`http://127.0.0.1:5173`

## 5. 页面说明

登录后，左侧导航可进入以下模块：

- 医生工作台：查看患者、病程时间线、T+1 预测和辅助建议

- 后端维护：查看患者总量、事件总量、关系分布、最近事件和最近更新患者

## 7. 常见问题

### 7.1 前端能打开，但没有患者数据

先确认：

- 后端是否已启动
- 浏览器是否访问的是 `127.0.0.1:5173`
- `/api/health` 是否正常返回

### 7.2 登录成功，但模型显示不可用

这表示系统接口可用，但 CTpath 权重或依赖没有成功加载。此时：

- 患者管理、时间线、维护页仍可正常使用
- 预测会优先走规则或相似病例回退策略

### 7.3 使用 MySQL 模式后看不到初始化病例

请确认：

1. 已执行 `app/mysql_schema.sql`
2. 已执行 `app/mysql_seed_demo.sql`
3. `CTPATH_DB_URL` 配置的是同一个库
4. 重启了 `uvicorn`

## 8. 推荐启动顺序

1. 先启动后端并确认 `api/health` 正常。
2. 再启动前端并使用演示账号登录。
3. 先在“后端维护”页确认患者数和事件数是否正常。
4. 再进入“医生工作台”做预测与联调。
