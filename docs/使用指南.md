# 系统使用指南

## 一、系统启动

### 1. 后端启动

```bash
# 安装依赖
pip install -r requirements-api.txt

# 启动后端服务
cd app
uvicorn main:app --reload --host 0.0.0.0 --port 8000
```

### 2. 前端启动

```bash
# 安装依赖
cd frontend
npm install

# 启动开发服务器
npm run dev
```

### 3. 访问系统

打开浏览器访问：http://localhost:5173

### 4. 登录记忆功能

系统提供登录记忆功能，方便多用户在同一设备上使用：

**功能特点**：
- ✅ 自动保存登录过的账号信息
- ✅ 最多保存10个账号
- ✅ 显示账号角色、部门、最后登录时间
- ✅ 快速选择账号登录
- ✅ 可移除单个账号或清空全部

**使用方法**：
1. 首次登录后，账号自动保存
2. 下次打开登录页面，显示已保存账号列表
3. 点击账号快速填充用户名
4. 输入密码后登录
5. 可移除不需要的账号

**安全说明**：
- 只保存账号基本信息（用户名、姓名、角色等）
- 不保存密码，每次登录需重新输入密码
- 数据保存在浏览器LocalStorage中
- 清除浏览器数据会删除保存的账号

## 三、功能说明

### 1. 医生工作台

**适用角色**：医生、管理员

**功能列表**：
- **患者队列**：查看待处理患者列表
- **搜索筛选**：按姓名、ID、疾病搜索，按风险等级筛选
- **患者详情**：查看患者完整信息
- **预测分析**：运行CTpath模型预测病程演化
- **随访管理**：创建随访任务、记录联系日志
- **档案管理**：编辑患者档案、补录诊疗事件

**操作流程**：
1. 登录后自动进入医生工作台
2. 在患者队列中选择患者
3. 点击"运行预测"查看预测结果
4. 根据预测结果制定诊疗计划
5. 安排随访任务

### 2. 护士工作台

**适用角色**：护士

**功能列表**：
- **今日随访任务**：查看今日待执行的随访任务
- **待处理任务**：查看所有待处理任务
- **患者查看**：查看患者基本信息（只读）
- **任务完成**：标记随访任务为已完成

**权限限制**：
- ❌ 不能运行预测
- ❌ 不能编辑患者档案
- ❌ 不能创建新患者
- ✅ 只能查看患者信息
- ✅ 只能执行随访任务

**操作流程**：
1. 登录后自动进入护士工作台
2. 查看今日随访任务列表
3. 点击"查看患者"了解患者情况
4. 执行随访（电话、门诊等）
5. 勾选完成任务

### 3. 档案管理

**适用角色**：档案管理员、医生

**功能列表**：
- **新建档案**：创建新的患者档案
- **批量导入**：从Excel导入患者数据
- **导出数据**：导出患者档案为JSON
- **编辑档案**：修改患者基本信息
- **补录事件**：添加诊疗事件到时间线
- **数据质量提示**：查看低支持度患者、缺失信息统计

**权限限制**：
- ❌ 不能运行预测
- ❌ 不能管理随访任务
- ✅ 只能管理患者档案

**操作流程**：
1. 点击"档案管理"进入档案模块
2. 选择"新建档案"或"批量导入"
3. 填写患者基本信息
4. 补充诊疗事件
5. 保存档案

### 4. 随访管理

**适用角色**：医生、护士

**功能列表**：
- **随访任务列表**：查看所有随访任务
- **流程看板**：查看患者处理流程
- **联系记录**：记录电话、门诊等联系结果
- **任务状态更新**：标记任务完成或关闭

## 四、数据说明

### 1. 患者数据

**基本信息**：
- 患者ID、姓名、年龄、性别
- 联系电话、紧急联系人
- 主诊断、当前阶段、风险等级

**诊疗信息**：
- 时序四元组：(主体, 关系, 客体, 时间戳)
- 关系类型：has_disease, stage, med_adherence, support_system等
- 时间线：按时间排序的诊疗事件

### 2. 预测结果

**输出内容**：
- Top-K预测：最可能的后续状态
- 置信度：预测结果的可信程度
- 证据路径：支持预测的关键事件
- 建议：基于预测结果的诊疗建议

### 3. 风险等级

| 等级 | 说明 | 颜色 |
|------|------|------|
| 高风险 | 需要优先关注和干预 | 红色 |
| 中风险 | 需要定期随访 | 橙色 |
| 低风险 | 病情稳定 | 绿色 |

## 五、API接口

### 1. 分页API

**接口**：`GET /api/patients/paginated`

**参数**：
- `page`: 页码（从1开始）
- `page_size`: 每页数量（最大100）
- `search`: 搜索关键词
- `risk_level`: 风险等级筛选
- `disease`: 疾病筛选

**响应**：
```json
{
  "patients": [...],
  "total": 156,
  "page": 1,
  "page_size": 20,
  "total_pages": 8
}
```

### 2. 预测API

**接口**：`POST /api/predict`

**请求**：
```json
{
  "patientId": "P001",
  "asOfTime": "2024-04-04",
  "topk": 5
}
```

**响应**：
```json
{
  "patientId": "P001",
  "predictions": [
    {
      "label": "Late",
      "score": 0.75,
      "reason": "..."
    }
  ],
  "evidence": {...}
}
```

## 六、常见问题

### 1. 登录失败

**原因**：
- 用户名或密码错误
- 账号未激活

**解决**：
- 检查用户名和密码
- 联系管理员激活账号

### 2. 权限不足

**原因**：
- 当前角色无该功能权限

**解决**：
- 切换到有权限的账号
- 联系管理员分配权限

### 3. 预测失败

**原因**：
- 患者数据不足
- 模型服务未启动

**解决**：
- 补充患者诊疗事件
- 检查模型服务状态

### 4. 数据加载慢

**原因**：
- 数据量大
- 网络延迟

**解决**：
- 使用分页功能
- 使用搜索筛选减少数据量
