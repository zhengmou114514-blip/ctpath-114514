# 启动与演示说明

## 启动顺序

### 1. 激活环境

```powershell
conda activate ctpath
cd E:\CTpath-master
```

### 2. 导入 MySQL 表结构与演示数据

```powershell
Get-Content .\app\mysql_schema.sql | mysql -u root -p ctpath
Get-Content .\app\mysql_seed_demo.sql | mysql -u root -p ctpath
```

### 3. 配置数据库环境变量

```powershell
$env:CTPATH_DB_URL="mysql+pymysql://root:你的密码@127.0.0.1:3306/ctpath?charset=utf8mb4"
```

### 4. 启动 FastAPI

```powershell
uvicorn app.main:app --host 127.0.0.1 --port 8000 --reload
```

### 5. 启动 Vue 前端

```powershell
cd E:\CTpath-master\frontend
cmd /c npm run dev
```

说明：

- 当前前端已改为使用 `vite.config.mjs` + `--configLoader native`
- 这样可以绕过 Windows 环境下 `vite.config.ts` 与 `esbuild` 加载时的 `EPERM / spawn EPERM` 问题
- 如果仍有缓存问题，可先删除 `frontend/node_modules/.vite-temp` 后再启动

## 演示检查清单

### 后端检查

```powershell
curl http://127.0.0.1:8000/api/health -UseBasicParsing
```

期望：

- `status = ok`
- `model_available = true`
- `mode = mysql` 为数据库演示
- `mode = demo` 为内置演示

### 登录并获取 token

```powershell
$login = Invoke-RestMethod -Method POST http://127.0.0.1:8000/api/login `
  -ContentType "application/json" `
  -Body '{"username":"demo_clinic","password":"demo123456"}'

$token = $login.token
```

### 验证患者详情

```powershell
Invoke-RestMethod http://127.0.0.1:8000/api/patient/PID9001 `
  -Headers @{ Authorization = "Bearer $token" }
```

### 验证时间线

```powershell
Invoke-RestMethod http://127.0.0.1:8000/api/timeline/PID9001 `
  -Headers @{ Authorization = "Bearer $token" }
```

### 验证预测

```powershell
Invoke-RestMethod -Method POST http://127.0.0.1:8000/api/predict `
  -Headers @{ Authorization = "Bearer $token" } `
  -ContentType "application/json" `
  -Body '{"patientId":"PID9001","topk":3}'
```

## 浏览器演示流程

3. 检索 `PID9001` 或 `PID0248`
